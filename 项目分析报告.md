## 项目概述

### 基本信息

- **项目名称**：jj.js
- **版本**：0.20.0
- **定位**：轻量级 Node.js MVC 框架
- **设计理念**：模仿 ThinkPHP5，基于 Proxy 实现类库自动加载、懒加载和 Class 自动实例化及单例化技术，采用单应用无限子级架构
- **开源协议**：MIT
- **作者**：yafoo
- **Node.js 要求**：>= 18

### 核心特性

1. **自动加载机制**：系统类库、用户类库都支持自动加载、懒加载、Class 自动生成单实例
2. **三级中间件**：支持应用级、路由级、控制器级三级中间件，方便插件及二次开发
3. **单应用多级模式**：单应用架构，`app/` 目录下支持无限子级分层（如 `app/admin/`、`app/api/`），子级自动向上继承父级类库
4. **完整的类型支持**：基于 JSDoc，提供完整的代码提示，自动生成应用端 types 类型文件（按目录层级生成）
5. **经典 MVC 架构**：系统架构为经典的 MVC 模式，很容易上手
6. **懒加载实例化**：Controller 和 Middleware 中的 View、Response 等实例采用懒加载模式，按需初始化

- 码云镜像：https://gitee.com/yafu/jj.js

## 核心原理

### 1. Proxy 自动加载机制

这是整个框架的**灵魂**，实现在 `lib/ctx.js` 和 `lib/loader.js` 中。

#### 1.1 工作原理

**核心入口文件 `jj.js`：**

```javascript
module.exports = new Proxy({}, {
    get: (target, prop) => {
        if(prop in target || typeof prop == 'symbol' || 
           ['inspect', 'router', 'run', 'types'].includes(prop)) {
            return target[prop];
        }
        prop = prop.toLowerCase();
        return require(`./lib/${prop == 'utils' ? 'utils/' : ''}${prop}`);
    }
});
```

这使得 `require('jj.js')` 可以按需加载任何模块：

```javascript
const {App, Controller, Db, Model, Logger} = require('jj.js');
```

#### 1.2 Ctx 类的 Proxy 拦截

`lib/ctx.js` 创建了所有业务类的基类，通过 Proxy 拦截 `$` 前缀的属性访问：

```javascript
const libs = ['ctx', 'context', 'controller', 'cache', 'cookie', 'db', 'logger', 
              'middleware', 'model', 'pagination', 'request', 'response', 
              'upload', 'url', 'view', 'utils', 'config'];

const Ctx = new Proxy(class Ctx {
    _setDeep(mod) {
        this.DEEP = pathToDeep(path.dirname(mod.path), cfg_app.base_dir);
    }
}, {
    construct() {
        return new Proxy({__proto__: arguments[2].prototype}, {
            get: (target, $prop, receiver) => {
                // 1. 如果属性已存在，直接返回
                if($prop in target || typeof $prop == 'symbol' || 
                   $prop == 'inspect' || $prop == 'ctx' || $prop == '$next' || $prop == 'DEEP') {
                    return Reflect.get(target, $prop, receiver);
                }

                const store = storage.getStore();
                
                // 2. 处理 $ 和 $$ 快捷访问
                if($prop == '$' || $prop == '$$') {
                    return store[$prop];
                }
                
                // 3. 只处理 $ 前缀的属性
                if($prop.slice(0, 1) != '$') {
                    return undefined;
                }

                const {$, $$, $config, DEEP} = store;
                const prop = $prop.slice(1);

                // 4. 根据 DEEP 向上遍历查找
                const DEEPS = (target.DEEP !== undefined ? target.DEEP : DEEP).split('/').filter(Boolean);
                const $nodes = [];

                DEEPS.reduce((node, deep) => {
                    if(!node) return;
                    if(node[deep] && node[deep][prop]) $nodes.unshift(node[deep][prop]);
                    return node[deep];
                }, $$);

                // 5. 添加配置和系统级类库
                if(prop == 'config' && $config) $nodes.push($config);
                if($$[prop]) $nodes.push($$[prop]);

                if(!libs.includes(prop) && !$nodes.length) return undefined;
                if(libs.includes(prop) && !$nodes.length) return $[prop];
                if(!libs.includes(prop) && $nodes.length == 1) return $nodes[0];

                // 6. 返回代理（支持 Class 实例化和属性访问）
                target[$prop] = new Proxy(function() {}, {
                    construct(_, args, newTarget) {
                        const node = $nodes.find(node => node.__ISCLASS__)
                        || libs.includes(prop) && $[prop].__ISCLASS__ && $[prop]
                        || undefined;
                        return node ? Reflect.construct(node, args, newTarget) : undefined;
                    },
                    get: (...args) => {
                        const node = $nodes.find(node => args[1] in node) || libs.includes(prop) && args[1] in $[prop] && $[prop] || {};
                        return Reflect.get(node, args[1], args[2]);
                    },
                    set: (...args) => {
                        const node = $nodes.find(node => args[1] in node);
                        if(node) {
                            node[args[1]] = args[2];
                        } else if(libs.includes(prop)) {
                            $[prop][args[1]] = args[2];
                        } else {
                            return false;
                        }
                        return true;
                    },
                });
                return Reflect.get(target, $prop, receiver);
            }
        });
    }
});
```

**查找顺序（向上继承）：**

```
this.$model.user （假设当前在 app/admin/ 子级）
    ↓
1. 检查当前子级 app/admin/model/user.js
    ↓ (不存在，向上查找父级)
2. 检查根级 app/model/user.js
    ↓ (不存在)
3. 检查 lib/ 目录下是否有对应系统类库
```

#### 1.3 Loader 类的智能加载

`lib/loader.js` 为目录/文件创建 Proxy 包装器，实现智能加载：

```javascript
function loader(path, ...args) {
    const dirPath = pt.normalize(path).replace(/[\/\\]+$/, '');
    const dirType = getNodeType(dirPath);
    if(dirType == '') return undefined;
    const node = getNode(dirType, dirPath);

    const infoStore = new Map();
    infoStore.set(node, {
        path: dirPath + pt.sep,
        type: dirType
    });
    return creatLoader(node);

    function creatLoader(node) {
        return new Proxy(node, {
            get: (node, prop, receiver) => {
                if(prop in node || typeof prop == 'symbol' || prop == 'inspect') {
                    return Reflect.get(node, prop, receiver);
                }

                const nodeInfo = infoStore.get(node);

                // Class 类型自动实例化（单例）
                if(nodeInfo.type == 'class') {
                    if(!nodeInfo.instance) {
                        nodeInfo.instance = new node(...args);
                    }
                    return prop in nodeInfo.instance 
                           ? nodeInfo.instance[prop] 
                           : prop == '__NODE__' ? nodeInfo 
                           : prop == '__ISCLASS__' ? true 
                           : nodeInfo.instance[prop];
                } else if(prop == '__NODE__') {
                    return nodeInfo;
                } else if(prop == '__ISCLASS__') {
                    return false;
                }

                // 查找子目录/文件
                const _childPath = nodeInfo.path + prop;
                const _childType = getNodeType(_childPath);
                if(_childType == '') return undefined;
                const _node = getNode(_childType, _childPath);
                infoStore.set(_node, {
                    path: _childPath + pt.sep,
                    type: _childType
                });
                
                // 递归创建 Proxy 并缓存
                return node[prop] = creatLoader(_node);
            },
            // ... set/has 拦截器
        });
    }
}

loader.clearPathCache = () => typeStore.clear();
```

**关键特性：**

- **类型智能识别**：自动识别 file/dir/json/class 四种节点类型
- **Class 单例化**：Class 类型自动 new 实例并缓存
- **路径缓存**：`typeStore` (Map) 缓存文件类型判断结果，避免重复检查
- **递归代理**：支持无限层级目录访问
- **clearPathCache**：提供清空路径缓存的方法

#### 1.4 链式调用支持

当直接调用类方法时（如 `this.$db.table('user')`），框架会自动检测：

1. 如果方法是静态方法，直接调用
2. 如果不是，自动 new 实例（单例），然后调用实例方法
3. 方法返回 `this`，支持链式调用

```javascript
// 示例
this.$db.table('user').where({status: 1}).select();
// 1. $db → 自动 new Db(ctx) 单例
// 2. table('user') → 返回 this
// 3. where({status: 1}) → 返回 this
// 4. select() → 执行查询，返回结果
```

### 2. AsyncLocalStorage 生命周期管理

使用 Node.js 的 `AsyncLocalStorage` 实现请求级别的数据隔离。

#### 2.1 实现位置

`lib/app.js` 中为每个请求创建独立的 storage：

```javascript
this.use(async (ctx, next) => {
    ctx.APP_TIME = Date.now();
    ctx.APP_VERSION = pjson.version;
    const _ = loader(cfg_app.base_dir, ctx);
    _.config || (_.config = {});
    Object.entries(config).forEach(([key, value]) => {
        _.config[key] = value;
    });
    await storage.run({
        $: loader(__dirname, ctx),       // 框架 lib 加载器
        $$: _.app,                        // 应用 app/ 目录加载器
        $config: _.config,                // 配置加载器
    }, async() => {
        await next();
    });
});
```

#### 2.2 生命周期

```
URL 请求到达
    ↓
创建 AsyncLocalStorage 上下文
    ├─ $: 框架 lib 加载器
    ├─ $$: 应用 app/ 目录加载器
    └─ $config: 配置加载器
    ↓
执行中间件链
    ↓
路由匹配（解析 DEEP/CONTROLLER/ACTION）
    ↓
执行控制器方法
    ├─ _init()
    ├─ action()
    └─ _end()
    ↓
请求结束，storage 自动清理
```

**优势：**

- 每个请求独立的类实例单例，互不干扰
- 避免并发请求间的状态污染
- 自动清理，无需手动管理
- 在异步调用链中保持上下文一致

### 3. 路由系统

基于 `@koa/router` 实现，支持自定义路由和默认路由。

#### 3.1 自定义路由

配置文件：`config/routes.js`

```javascript
routes = [
    // 基本路由
    {url: '/', path: 'app/index/index2'},
    
    // 动态参数 + 路由命名
    {url: '/article/:id.html', path: 'app/article/article', name: 'article'},
    
    // 中间件类型
    {url: '/admin', path: 'app/admin/check', type: 'middleware'},
    
    // 模板直接输出
    {url: '/about', path: 'app/about/index', type: 'view'},
    
    // 动态参数替换
    {url: '/${id}/detail', path: 'app/detail/show'},
    
    // 直接路由到函数
    {url: '/hello', path: async (ctx, next) => {
        ctx.body = 'hello world!';
    }}
];
```

#### 3.2 默认路由

```javascript
// 系统内置默认路由（单应用多级）
router.all('/(.*)', async (ctx, next) => {
    if(ctx._routeMatched) return await next();

    let param = ctx.params[0] ?? '';
    const dotIndex = param.indexOf('.');
    if(~dotIndex) param = param.slice(0, dotIndex);
    try {
        resolveRoute(param, ctx, cfg_app.controller_folder);
        delete ctx.params[0];
    } catch (error) {
        if(cfg_app.app_debug) throw error;
        return;
    }
    await next();
});

// resolveRoute 使用 pathToRouteInfo 解析路径
function resolveRoute(routePath, ctx, controllerFolder) {
    const routeInfo = pathToRouteInfo(routePath, cfg_app.default_deep, 
                                       cfg_app.default_controller, cfg_app.default_action);
    ctx.DEEP = routeInfo.DEEP;                   // 子级深度，如 'admin' 或 'api/v1'
    ctx.CONTROLLER_FOLDER = controllerFolder;
    ctx.CONTROLLER = routeInfo.CONTROLLER;
    ctx.ACTION = routeInfo.ACTION;
}
```

#### 3.3 路由反向编译

通过路由命名可以反向生成 URL：

```javascript
// 配置
{name: 'article', url: '/article/:id.html', path: 'app/article/article'}

// 使用
this.$url.build(':article', {id: 123});
// 输出：/article/123.html
```

### 4. MVC 中间件执行流程

`lib/mvc.js` 负责控制器的完整执行流程：

```javascript
async function mvc(ctx, next) {
    const controller_folder = ctx.CONTROLLER_FOLDER || cfg_app.controller_folder;

    // 1. 调试日志
    if(cfg_app.app_debug) {
        Logger.http({Router: controller_folder, DEEP: ctx.DEEP, CONTROLLER: ctx.CONTROLLER, ACTION: ctx.ACTION});
        Object.keys(ctx.params).length && Logger.http('Params:', ctx.params);
        Object.keys(ctx.query).length && Logger.http('Get:', ctx.query);
        Object.keys(ctx.request.body || {}).length && Logger.http('Post:', ctx.request.body);
    }
    
    // 2. 设置 storage 上下文
    const store = storage.getStore();
    store.DEEP = ctx.DEEP;
    store.CONTROLLER = ctx.CONTROLLER;
    store.ACTION = ctx.ACTION;

    // 3. 根据 DEEP 向上遍历查找控制器
    const controllerPaths = [...ctx.DEEP.split('/').filter(Boolean), controller_folder];
    const $node = controllerPaths.reduce((node, deep) => {
        if(!node[deep]) {
            throw new Error(`RunError: 目录:/app/${controllerPaths.join('/')}不存在！`);
        }
        return node[deep];
    }, store.$$);

    // 4. 实例化控制器
    const Controller = $node[ctx.CONTROLLER] || $node['_empty'];
    const $controller = new Controller(ctx, next);
    const camelCaseAction = toCamelCase(ctx.ACTION); // 转换为驼峰命名
    const action = typeof $controller[camelCaseAction] == 'function' 
                   && !$controller[camelCaseAction].__ISCLASS__
                   ? camelCaseAction 
                   : '_empty';

    // 5. 加载控制器中间件
    const middlewares = Array.isArray($controller['middleware'])
        && $controller['middleware'].reduce((stack, middle) => {
            // 过滤 except/accept 配置
            // 加载中间件类
            // ...
            return stack.concat(async (ctx, next) => {
                const {DEEPS, MIDDLEWARE, ACTION} = pathToMiddlewareInfo(
                    middle.middleware, ctx.DEEP, ctx.CONTROLLER, action
                );
                const $middleware = new Middleware(ctx, next);
                await $middleware[ACTION]();
            });
        }, []) || [];

    // 6. 执行控制器方法（含中间件）
    await compose(middlewares)(ctx, async () => {
        let result;
        typeof $controller['_init'] == 'function' 
            && (result = await $controller['_init']());
        result !== false 
            && (result = await $controller[action]());
        result !== false 
            && typeof $controller['_end'] == 'function' 
            && await $controller['_end']();
    });
}
```

**执行顺序：**

```
请求到达
    ↓
验证 DEEP/CONTROLLER/ACTION
    ↓
实例化控制器
    ↓
加载控制器中间件（middleware 数组）
    ↓
compose 中间件链
    ↓
执行 _init() (如果存在)
    ↓
    返回 false? ──→ 终止
    ↓ 否
执行 action()
    ↓
    返回 false? ──→ 终止
    ↓ 否
执行 _end() (如果存在)
    ↓
响应返回
```

### 1. App 类 (lib/app.js)

应用入口类，继承自 Koa，负责初始化和中间件注册。

```javascript
class App extends Koa {
    constructor(options) {
        // 支持多种参数形式
        if(typeof options == 'function') {
            options = { middleware: [options] };
        } else if(Array.isArray(options)) {
            options = { middleware: options };
        }
        super(options);
        this._init(options);
    }

    _init(options) {
        // 1. storage 中间件 - 生命周期管理
        this.use(async (ctx, next) => {
            ctx.APP_TIME = Date.now();
            ctx.APP_VERSION = pjson.version;
            const _ = loader(cfg_app.base_dir, ctx);
            _.config || (_.config = {});
            Object.entries(config).forEach(([key, value]) => {
                _.config[key] = value;
            });
            await storage.run({
                $: loader(__dirname, ctx),
                $$: _.app,
                $config: _.config,
            }, async() => {
                await next();
            });
        });

        // 2. 用户自定义中间件
        options && Array.isArray(options.middleware) && options.middleware.forEach(middleware => {
            this.use(middleware);
        });

        // 3. 异常处理中间件
        this.use(async (ctx, next) => {
            try {
                await next();
            } catch (err) {
                const errorMessage = err instanceof Error ? (err.stack || '') : String(err);
                errorMessage.split("\n").forEach(line => {
                    Logger.error(line);
                });
                if(cfg_app.app_debug) {
                    new Response(ctx).exception(err instanceof Error ? err : new Error(errorMessage));
                } else {
                    const errorStatus = typeof err === 'object' && err && (err.statusCode || err.status) || 500;
                    ctx.response.status = errorStatus;
                    ctx.body = 'Internal Server Error';
                }
            }
            const ms = Date.now() - ctx.APP_TIME;
            Logger.http(`${ctx.method} ${ctx.status} ${ctx.url} - ${ms}ms`);
        });

        // 4. 静态资源
        cfg_app.static_dir && this.use(require('koa-static')(
            path.join(cfg_app.base_dir, cfg_app.static_dir)
        ));

        // 5. 请求体解析
        cfg_app.koa_body && this.use(require('koa-body').koaBody(cfg_app.koa_body));

        // 6. 路由
        this.use(router.routes()).use(router.allowedMethods());

        // 7. MVC 中间件
        this.use(mvc);

        // 8. Cookie 密钥
        cfg_cookie.keys && (this.keys = cfg_app.app_debug 
                            ? ['jj.js'] 
                            : cfg_cookie.keys);

        // 9. 自动生成 types 文件（调试模式）
        if(cfg_app.app_debug) {
            const jsconfigFile = path.join(cfg_app.base_dir, 'jsconfig.json');
            if(!isFileSync(jsconfigFile)) Logger.warning('jsconfig.json not found!');
            require('./types').watch();
        }
    }
}
```

```javascript
class Model extends Context {
    constructor(ctx) {

        this.connection = null;
        this.table = toUnderScore(this.constructor.name); // 自动设置表名
        this.pk = 'id';  // 主键

                delete data[this.pk];

        delete data[this.pk];
        return await this.add(data);
```

### 5. Cache 类 (lib/cache.js)

静态缓存类，支持自动过期清理。

```javascript
class Cache {
    constructor() {
        if(new.target) {
            class ChildCache extends this.constructor {}
            ChildCache.cache = {};
            ChildCache.timer = null;
            ChildCache.setIntervalTime(cfg_cache.clear_time);
            return ChildCache;
        }
    }

    static get(key) {
        if(key === undefined) return this.cache;
        
        const now_time = Math.round(Date.now() / 1000);
        if(this.cache[key] && this.cache[key].time > now_time) {
            return this.cache[key].data;
        } else {
            this.delete(key);
            return undefined;
        }
    }

    static set(key, data, cache_time) {
        cache_time = cache_time || cfg_cache.cache_time 
                     || 60 * 60 * 24 * 365 * 10; // 默认 10 年
        const now_time = this.time();
        this.cache[key] = {
            data: data, 
            time: cache_time + now_time
        };
    }

    static delete(key) {
        if(key) {
            delete this.cache[key];
        } else {
            this.cache = {};
        }
    }

    static setIntervalTime(time) {
        if(time) {
            this.timer && clearInterval(this.timer);
            this.timer = setInterval(() => {
                const now_time = this.time();
                for(let key in this.cache) {
                    this.cache[key].time < now_time && this.delete(key);
                }
            }, time * 1000);
        } else {
            this.timer && clearInterval(this.timer);
            this.timer = null;
        }
    }

    static time() {
        return Math.round(Date.now() / 1000);
    }
}

// 初始化
Cache.cache = {};
Cache.timer = null;
Cache.setIntervalTime(cfg_cache.clear_time);
```


### 7. Url 类 (lib/url.js)

URL 生成类，支持智能解析和路由反向编译。

```javascript
class Url extends Context {
    // 编译生成 URL
    build(url='', vars={}, ext='', domain='') {
        // 参数处理
        if(typeof vars != 'object') {
            [vars, ext, domain] = [{}, vars, ext];
        }
        if(ext === true || ext.slice(0, 4) == 'http') {
            [ext, domain] = ['', ext];
        }
        if(domain === true) {
            domain = this.ctx.protocol + '://' + this.ctx.host;
        }

        const query = {...vars};
        let urls;

        // 1. 路由命名（:name 格式）
        if(url.slice(0, 1) == ':') {
            url = this.ruleUrl(url.slice(0, 1), query);
        }
        // 2. 相对路径智能解析
        else if(url.slice(0, 1) != '/' && url.slice(0, 4) != 'http') {
            urls = url.split('/').reverse().map(u => toUnderScore(u));
            if(urls.length < 3) {
                url = '/' + (this.ctx.DEEP ? this.ctx.DEEP + '/' : '')
                      + (urls[1] || this.ctx.CONTROLLER) 
                      + '/' + (urls[0] || this.ctx.ACTION);
            } else {
                url = '/' + urls.reverse().join('/');
            }
        }

        // 3. 添加参数和后缀
        urls = url.split('#');
        if(~urls[0].indexOf('?')) {
            const urlss = urls[0].split('?');
            urlss[0] += ext;
            if(Object.keys(query).length > 0) {
                urlss[1] = querystring.stringify({
                    ...querystring.parse(urlss[1]), 
                    ...query
                });
            }
            urls[0] = urlss.join('?');
        } else {
            urls[0] = urls[0] + ext;
            if(Object.keys(query).length > 0) {
                urls[0] = urls[0] + '?' + querystring.stringify(query);
            }
        }

        return domain + urls.join('#');
    }

    // 路由反向编译
    ruleUrl(url, query = {}) {
        if(!routes[url]) return url;
        url = routes[url].url;

        if(~url.indexOf(':')) {
            url = url.replace(/\:([^/]+\(.*\)|[^/]+)/g, (match, key) => {
                key = key.split(/[(.]/);
                const k = key[0];
                if(k in query) {
                    const value = query[k];
                    delete query[k];
                    return querystring.escape(value) 
                           + (!key[1] || ~key[1].indexOf(')') ? '' : '.' + key[1]);
                } else if(this.ctx.params && k in this.ctx.params) {
                    return querystring.escape(this.ctx.params[k]);
                }
                return match;
            });
        }

        return url;
    }
}
```

```javascript
// → /user/list

// 路由命名
```

## 工作流程

### 完整请求流程

```
1. 用户发起 HTTP 请求
   ↓
2. Koa 接收请求，进入中间件链
   ↓
3. Storage 中间件
   - 记录 APP_TIME、APP_VERSION
   - 创建 AsyncLocalStorage 上下文
   - 初始化 $、$$、$config 加载器
   ↓
4. 用户自定义中间件（如果有）
   ↓
5. 异常处理中间件
   - try { await next() }
   - catch 错误并记录日志
   - 调试模式输出详细错误
   - 记录请求日志（方法、状态、URL、耗时）
   ↓
6. 静态资源中间件（如果配置）
   - 匹配 public 目录文件
   - 匹配则直接返回，不继续
   ↓
7. koa-body 中间件（如果配置）
   - 解析 POST 数据
   - 处理文件上传
   ↓
8. 路由中间件
   - 匹配自定义路由（routes.js）
   - 匹配默认路由（单应用多级解析）
   - 设置 ctx.DEEP、ctx.CONTROLLER_FOLDER、ctx.CONTROLLER、ctx.ACTION
   ↓
9. mvc() 执行
   - 设置 storage 上下文（DEEP、CONTROLLER、ACTION）
   - 根据 DEEP 向上遍历查找控制器
   ↓
10. 控制器处理
    - 实例化控制器类
    - 加载控制器中间件
    - 执行 _init()
    - 执行 action()
    - 执行 _end()
    ↓
11. 响应输出
    - $show() 直接输出
    - $fetch() 渲染模板
    - $redirect() 重定向
    - $success()/$error() 提示跳转
    ↓
12. 中间件链返回（洋葱模型）
    ↓
13. 请求结束，AsyncLocalStorage 自动清理
```

## 🔄 近期重要变更（v0.20.0）

### 核心架构变更

#### 1. 单/多应用模式重构为单应用无限子级模式

- **变更前**：支持单应用和多应用两种模式，通过 `app_multi` 配置切换
- **变更后**：统一为单应用架构，`app/` 目录下支持无限子级分层
- **影响**：
  - 删除 `app_multi` 配置项
  - 删除 `common_app` 配置项
  - `default_app` 改为 `default_deep`
  - 新增 `middleware_folder` 配置项
  - 不再有 `common/` 公共应用目录和独立应用目录

#### 2. Ctx 类 Proxy 重构（向上继承机制）

**变更内容：**
- `_` 加载器改为 `$$`（应用目录加载器）
- `APP` 改为 `DEEP`（子级深度）
- `COMMON` 公共应用查找改为 DEEP 向上遍历
- 新增 `$config` 独立配置加载器
- 新增 `DEEP` 属性直接访问支持
- 新增 `set` 拦截器

```javascript
// 变更前
const {$, _, APP, COMMON} = store;
// 查找：当前应用 → 公共应用 → 系统类库

// 变更后
const {$, $$, $config, DEEP} = store;
// 查找：当前子级 → 父级（向上遍历） → 根级 → 系统类库
```

#### 3. 路由系统重构

- `ctx.APP` → `ctx.DEEP`
- `ctx.FOLDER` → `ctx.CONTROLLER_FOLDER`
- 路由解析使用 `pathToRouteInfo()` 函数，支持多级路径解析
- 删除多应用路由，统一为单应用多级路由

#### 4. MVC 中间件重构

- `run()` 函数改为 `mvc()` 中间件
- 控制器查找改为根据 DEEP 向上遍历 `$$` 加载器
- `toHump()` 改为 `toCamelCase()`
- 中间件查找使用 `pathToMiddlewareInfo()` 函数

#### 5. 内置函数重命名

- `toHump()` → `toCamelCase()`
- `toLine()` → `toUnderScore()`

#### 6. Types 生成器重构

- 移除 `watch` 依赖，改用 Node.js 原生 `fs.watch`
- 重构为 `TypesGenerator` 类
- 新增防抖机制（300ms）
- 支持按目录层级生成类型文件
- 子级 Ctx 继承父级 $ 属性
- 监控范围限定为 `app/` 目录

#### 7. Db 类变更

- 查询方法不再返回 SQL 字符串，统一通过 `db.sql` 属性获取
- 移除 `getCache()`/`setCache()`/`deleteCache()` 方法
- `cache(time)` 改为 `withCache(time)`
- `pagination()` 改为 `paginate()`
- 默认数据库改为 SQLite

#### 8. Loader 类变更

- `__node` 重命名为 `__NODE__`
- 新增 `__ISCLASS__` 属性
- `loader` 函数参数 `dir` 改名为 `path`，要求绝对路径
- 新增 `clearPathCache()` 方法
- 新增 JSON 文件加载支持

   - 明确的系统类库白名单（libs 数组）
   - 向上继承机制，子级自动复用父级类库

   - 默认路由（智能解析）
   - 路由命名和反向编译
   - 支持动态参数
   - 支持三种路由类型（controller/middleware/view）

   - 调试模式友好（详细的 HTTP 日志）

   - 按需加载模块
   - 静态资源可选
   - View/Response 实例按需创建

### 技术亮点

| 特性 | 实现方式 | 优势 |
|------|---------|------|
| 自动加载 | Proxy 拦截 + DEEP 向上遍历 | 无需手动导入，子级继承父级 |
| 生命周期管理 | AsyncLocalStorage | 请求隔离，自动清理 |
