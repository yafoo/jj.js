## 项目概述

### 基本信息

- **项目名称**：jj.js
- **版本**：0.19.0
- **定位**：轻量级 Node.js MVC 框架
- **设计理念**：模仿 ThinkPHP5，基于 Proxy 实现类库自动加载、懒加载和 Class 自动实例化及单例化技术
- **开源协议**：MIT
- **作者**：yafoo
- **Node.js 要求**：>= 18

### 核心特性

1. **自动加载机制**：系统类库、用户类库都支持自动加载、懒加载、Class 自动生成单实例
2. **三级中间件**：支持应用级、路由级、控制器级三级中间件，方便插件及二次开发
3. **多应用模式**：支持单应用和多应用两种运行模式
4. **完整的类型支持**：基于 JSDoc，提供完整的代码提示，自动生成应用端 types 类型文件
5. **经典 MVC 架构**：系统架构为经典的 MVC 模式，很容易上手

- 码云镜像：https://gitee.com/yafu/jj.js

## 核心原理

### 1. Proxy 自动加载机制

这是整个框架的**灵魂**，实现在 `lib/ctx.js` 和 `lib/loader.js` 中。

#### 1.1 工作原理

**核心入口文件 `jj.js`：**

```javascript
module.exports = new Proxy({}, {
    get: (target, prop) => {
        if(prop in target || typeof prop == 'symbol' || 
           ['inspect', 'router', 'run', 'types'].includes(prop)) {
            return target[prop];
        }
        prop = prop.toLowerCase();
        return require(`./lib/${prop == 'utils' ? 'utils/' : ''}${prop}`);
    }
});
```

这使得 `require('jj.js')` 可以按需加载任何模块：

```javascript
const {App, Controller, Db, Model, Logger} = require('jj.js');
```

#### 1.2 Ctx 类的 Proxy 拦截

`lib/ctx.js` 创建了所有业务类的基类，通过 Proxy 拦截 `$` 前缀的属性访问：

```javascript
const Ctx = new Proxy(class {}, {
    construct() {
        return new Proxy({__proto__: arguments[2].prototype}, {
            get: (target, $prop, receiver) => {
                // 1. 如果属性已存在，直接返回
                if($prop in target || typeof $prop == 'symbol' || $prop == 'inspect') {
                    return Reflect.get(target, $prop, receiver);
                }

                const store = storage.getStore();
                const $ = store.$;      // 框架 lib 加载器
                const _ = store._;      // 应用目录加载器
                
                // 2. 只处理 $ 前缀的属性
                if($prop.slice(0, 1) != '$') {
                    return undefined;
                }

                const prop = $prop.slice(1);
                const APP = store.APP;
                const COMMON = store.COMMON;

                // 3. 缓存机制：第一次查找后缓存结果
                if(_[APP] && _[APP][$prop] === undefined) {
                    // 查找顺序：
                    // a. 应用目录优先
                    // b. 公共应用目录
                    // c. 框架 lib 目录
                    if(_[APP][prop] && _[COMMON] && _[COMMON][prop]) {
                        _[APP][$prop] = new Proxy({}, {
                            get: (...args) => {
                                return _[APP][prop][args[1]] || _[COMMON][prop][args[1]];
                            }
                        });
                    } else if(_[APP][prop]) {
                        _[APP][$prop] = _[APP][prop];
                    } else if(_[COMMON] && _[COMMON][prop]) {
                        _[APP][$prop] = _[COMMON][prop];
                    } else {
                        _[APP][$prop] = null;
                    }
                }

                // 4. 返回找到的模块
                return _[APP] && _[APP][$prop] || _[prop] || 
                       (!['app', 'loader', 'router', 'run', 'storage', 'types'].includes(prop) && $[prop]) || 
                       undefined;
            }
        });
    }
});
```

**查找顺序：**

```
this.$model.user
    ↓
1. 检查 app/model/user.js (当前应用)
    ↓ (不存在)
2. 检查 common/model/user.js (公共应用)
    ↓ (不存在)
3. 检查 lib/ 目录下是否有对应模块
```

#### 1.3 Loader 类的智能加载

`lib/loader.js` 为目录/文件创建 Proxy 包装器，实现智能加载：

```javascript
function loader(dir='./', ...args) {
    // 1. 确定目录类型
    dir = pt.isAbsolute(dir) ? dir : pt.join(pt.dirname(module.parent.filename), dir);
    const dirPath = pt.join(dir, './');
    let dirType = isFileSync(dir + '.js') ? 'file' 
                 : isFileSync(dir + '.json') ? 'json' 
                 : 'dir';
    
    const node = dirType == 'dir' ? {} : require(dir);
    
    // 2. 如果是 Class，标记为 class 类型
    if(dirType == 'file' && isClass(node)) {
        dirType = 'class';
    }

    // 3. 创建 Proxy 代理
    return creatLoader(node);

    function creatLoader(node) {
        return new Proxy(node, {
            get: (node, prop, receiver) => {
                // 如果属性已存在，直接返回
                if(prop in node || typeof prop == 'symbol' || prop == 'inspect') {
                    return Reflect.get(node, prop, receiver);
                }

                const nodeInfo = infoStore.get(node);

                // 4. Class 类型自动实例化（单例）
                if(nodeInfo.type == 'class') {
                    if(!nodeInfo.instance) {
                        nodeInfo.instance = new node(...args);
                    }
                    return prop in nodeInfo.instance 
                           ? nodeInfo.instance[prop] 
                           : (prop == '__node' ? nodeInfo : nodeInfo.instance[prop]);
                }

                // 5. 查找子目录/文件
                let _nodeType = typeStore[_nodePath];
                if(!typeStore[_nodePath]) {
                    // 文件检查顺序：.js → .json → 目录
                    _nodeType = isFileSync(_nodeFile) ? 'file' 
                               : isFileSync(_nodeJson) ? 'json' 
                               : isDirSync(_nodePath) ? 'dir'
                               : '';
                    typeStore[_nodePath] = _nodeType; // 缓存结果
                }

                // 6. 根据类型加载
                if(_nodeType == 'file' || _nodeType == 'class') {
                    _node = require(_nodeFile);
                } else if(_nodeType == 'json') {
                    _node = require(_nodeJson);
                }

                // 7. 递归创建 Proxy
                node[prop] = creatLoader(_node);
                return node[prop];
            }
        });
    }
}
```

**关键特性：**

- **类型智能识别**：自动识别文件/目录/JSON/Class
- **Class 单例化**：Class 类型自动 new 实例并缓存
- **路径缓存**：`typeStore` 缓存文件类型判断结果，避免重复检查
- **递归代理**：支持无限层级目录访问

#### 1.4 链式调用支持

当直接调用类方法时（如 `this.$db.table('user')`），框架会自动检测：

1. 如果方法是静态方法，直接调用
2. 如果不是，自动 new 实例（单例），然后调用实例方法
3. 方法返回 `this`，支持链式调用

```javascript
// 示例
this.$db.table('user').where({status: 1}).select();
// 1. $db → 自动 new Db(ctx) 单例
// 2. table('user') → 返回 this
// 3. where({status: 1}) → 返回 this
// 4. select() → 执行查询，返回结果
```

### 2. AsyncLocalStorage 生命周期管理

使用 Node.js 的 `AsyncLocalStorage` 实现请求级别的数据隔离。

#### 2.1 实现位置

`lib/app.js` 中为每个请求创建独立的 storage：

```javascript
this.use(async (ctx, next) => {
    ctx.APP_TIME = Date.now();
    ctx.APP_VERSION = pjson.version;
    
    await storage.run({
        $: loader('./', ctx),              // 框架 lib 加载器
        _: loader(cfg_app.base_dir, ctx)   // 应用目录加载器
    }, async() => {
        await next();
    });
});
```

#### 2.2 生命周期

```
URL 请求到达
    ↓
创建 AsyncLocalStorage 上下文
    ├─ $: 框架 lib 加载器
    └─ _: 应用目录加载器
    ↓
执行中间件链
    ↓
路由匹配
    ↓
执行控制器方法
    ├─ _init()
    ├─ action()
    └─ _end()
    ↓
请求结束，storage 自动清理
```

**优势：**

- 每个请求独立的类实例单例，互不干扰
- 避免并发请求间的状态污染
- 自动清理，无需手动管理
- 在异步调用链中保持上下文一致

### 3. 路由系统

基于 `@koa/router` 实现，支持自定义路由和默认路由。

#### 3.1 自定义路由

配置文件：`config/routes.js`

```javascript
routes = [
    // 基本路由
    {url: '/', path: 'app/index/index2'},
    
    // 动态参数 + 路由命名
    {url: '/article/:id.html', path: 'app/article/article', name: 'article'},
    
    // 中间件类型
    {url: '/admin', path: 'app/admin/check', type: 'middleware'},
    
    // 模板直接输出
    {url: '/about', path: 'app/about/index', type: 'view'},
    
    // 动态参数替换
    {url: '/${id}/detail', path: 'app/detail/show'},
    
    // 直接路由到函数
    {url: '/hello', path: async (ctx, next) => {
        ctx.body = 'hello world!';
    }}
];
```

#### 3.2 默认路由

```javascript
// 单应用模式
router.all('/:CONTROLLER?/:ACTION?', async (ctx, next) => {
```

### 4. 控制器执行流程

`lib/run.js` 负责控制器的完整执行流程：

```javascript
async function run(ctx, next, control_type='controller') {
    // 1. 设置 storage 上下文
    const store = storage.getStore();
    store.APP = ctx.APP;
    store.CONTROLLER = ctx.CONTROLLER;
    store.ACTION = ctx.ACTION;
    store.COMMON = cfg_app.common_app;

    // 2. 加载配置
    store._.config === undefined && (store._.config = {});
    Object.entries(config).forEach(([key, value]) => {
        store._.config[key] = value;
    });

    // 3. 验证应用/控制器/方法
    // ... 验证逻辑

    // 4. 实例化控制器
    const $controller = new Controller(ctx, next);
    const humpAction = toHump(ctx.ACTION); // 转换为驼峰命名
    const action = typeof $controller[humpAction] == 'function' 
                   ? humpAction 
                   : '_empty';

    // 5. 加载控制器中间件
    const middlewares = Array.isArray($controller['middleware'])
        && $controller['middleware'].reduce((stack, middle) => {
            // 过滤 except/accept 配置
            // 加载中间件类
            // ...
            return stack.concat(async (ctx, next) => {
                const $middleware = new Middleware(ctx, next);
                await $middleware[METHOD]();
            });
        }, []) || [];

    // 6. 执行控制器方法（含中间件）
    await compose(middlewares)(ctx, async () => {
        let result;
        typeof $controller['_init'] == 'function' 
            && (result = await $controller['_init']());
        result !== false 
            && (result = await $controller[action]());
        result !== false 
            && typeof $controller['_end'] == 'function' 
            && await $controller['_end']();
    });
}
```

**执行顺序：**

```
请求到达
    ↓
验证 APP/CONTROLLER/ACTION
    ↓
实例化控制器
    ↓
加载控制器中间件（middleware 数组）
    ↓
compose 中间件链
    ↓
执行 _init() (如果存在)
    ↓
    返回 false? ──→ 终止
    ↓ 否
执行 action()
    ↓
    返回 false? ──→ 终止
    ↓ 否
执行 _end() (如果存在)
    ↓
响应返回
```

### 5. 数据库查询构造器

`lib/db.js` 实现了完整的查询构造器，支持链式调用。

#### 5.1 查询参数构建

```javascript
reset() {
    this._options = {
        distinct: '',          // DISTINCT 关键字

        where: [],             // 查询条件
```

#### 5.2 链式调用示例

```javascript
this.$db
```

#### 5.3 WHERE 条件解析

支持丰富的查询条件：

```javascript
// 简单条件
.where({name: '张三'})
// → `name` = ?

// 比较运算符
.where({age: ['>', 18]})
// → `age` > ?

.where({score: ['between', [60, 100]]})
// → `score` BETWEEN ? AND ?

.where({name: ['like', '%张%']})
// → `name` LIKE ?

.where({id: ['in', [1, 2, 3]]})
// → `id` IN (?)

// 逻辑关系
.where({status: 1}, 'or')
// → OR (`status` = ?)
```

```javascript
class App extends Koa {
    constructor(options) {
        // 支持多种参数形式
        if(typeof options == 'function') {
            options = { middleware: [options] };
        } else if(Array.isArray(options)) {
            options = { middleware: options };
        }
        super(options);
        this._init(options);
    }

    _init(options) {
        // 1. storage 中间件 - 生命周期管理
        this.use(async (ctx, next) => {
            ctx.APP_TIME = Date.now();
            ctx.APP_VERSION = pjson.version;
            await storage.run({
                $: loader('./', ctx),
                _: loader(cfg_app.base_dir, ctx)
            }, async() => {
                await next();
            });
        });

        // 2. 用户自定义中间件
        options && options.middleware && options.middleware.forEach(middleware => {
            this.use(middleware);
        });

        // 3. 异常处理中间件
        this.use(async (ctx, next) => {
            try {
                await next();
            } catch (err) {
                Logger.error(...err.stack.split("\n"));
                if(cfg_app.app_debug) {
                    new Response(ctx).exception(err);
                } else {
                    ctx.response.status = err.statusCode || err.status || 500;
                    ctx.body = 'Internal Server Error';
                }
            }
            const ms = Date.now() - ctx.APP_TIME;
            Logger.http(`${ctx.method} ${ctx.status} ${ctx.url} - ${ms}ms`);
        });

        // 4. 静态资源
        cfg_app.static_dir && this.use(require('koa-static')(
            path.join(cfg_app.base_dir, cfg_app.static_dir)
        ));

        // 5. 请求体解析
        cfg_app.koa_body && this.use(require('koa-body').koaBody(cfg_app.koa_body));

        // 6. 路由
        this.use(router.routes()).use(router.allowedMethods());

        // 7. Cookie 密钥
        cfg_cookie.keys && (this.keys = cfg_app.app_debug 
                            ? ['jj.js'] 
                            : cfg_cookie.keys);

        // 8. 自动生成 types 文件（调试模式）
        if(cfg_app.app_debug) {
            const jsconfigFile = path.join(cfg_app.base_dir, 'jsconfig.json');
            isFileSync(jsconfigFile) && require('./types')();
        }
    }
```

### 5. Cache 类 (lib/cache.js)

```javascript
class Cache {
    constructor() {
        if(new.target) {
            class ChildCache extends this.constructor {}
            ChildCache.cache = {};
            ChildCache.timer = null;
            ChildCache.setIntervalTime(cfg_cache.clear_time);
            return ChildCache;
        }
    }

    static get(key) {
        if(key === undefined) return this.cache;
        
        const now_time = Math.round(Date.now() / 1000);
        if(this.cache[key] && this.cache[key].time > now_time) {
            return this.cache[key].data;
        } else {
            this.delete(key);
            return undefined;
        }
    }

    static set(key, data, cache_time) {
        cache_time = cache_time || cfg_cache.cache_time 
                     || 60 * 60 * 24 * 365 * 10; // 默认 10 年
        const now_time = this.time();
        this.cache[key] = {
            data: data, 
            time: cache_time + now_time
        };
    }

    static delete(key) {
        if(key) {
            delete this.cache[key];
        } else {
            this.cache = {};
        }
    }

    static setIntervalTime(time) {
        if(time) {
            this.timer && clearInterval(this.timer);
            this.timer = setInterval(() => {
                const now_time = this.time();
                for(let key in this.cache) {
                    this.cache[key].time < now_time && this.delete(key);
                }
            }, time * 1000);
        } else {
            this.timer && clearInterval(this.timer);
            this.timer = null;
        }
    }

    static time() {
        return Math.round(Date.now() / 1000);
    }
}
```

### 7. Url 类 (lib/url.js)

URL 生成类，支持智能解析和路由反向编译。

```javascript
class Url extends Context {
    // 编译生成 URL
    build(url='', vars={}, ext='', domain='') {
        // 参数处理
        if(typeof vars != 'object') {
            [vars, ext, domain] = [{}, vars, ext];
        }
        if(ext === true || ext.slice(0, 4) == 'http') {
            [ext, domain] = ['', ext];
        }
        if(domain === true) {
            domain = this.ctx.protocol + '://' + this.ctx.host;
        }

        const query = {...vars};
        let urls;

        // 1. 路由命名（:name 格式）
        if(url.slice(0, 1) == ':') {
            url = this.ruleUrl(url.slice(0, 1), query);
        }
        // 2. 相对路径智能解析
        else if(url.slice(0, 1) != '/' && url.slice(0, 4) != 'http') {
            urls = url.split('/').reverse().map(u => toLine(u));
            if(urls.length < 3) {
                url = ((cfg_app.app_multi && '/' + this.ctx.APP) || '') 
                      + '/' + (urls[1] || this.ctx.CONTROLLER) 
                      + '/' + (urls[0] || this.ctx.ACTION);
            } else {
                url = '/' + urls.reverse().join('/');
            }
        }

        // 3. 添加参数和后缀
        urls = url.split('#');
        if(~urls[0].indexOf('?')) {
            const urlss = urls[0].split('?');
            urlss[0] += ext;
            if(Object.keys(query).length > 0) {
                urlss[1] = querystring.stringify({
                    ...querystring.parse(urlss[1]), 
                    ...query
                });
            }
            urls[0] = urlss.join('?');
        } else {
            urls[0] = urls[0] + ext;
            if(Object.keys(query).length > 0) {
                urls[0] = urls[0] + '?' + querystring.stringify(query);
            }
        }

        return domain + urls.join('#');
    }

    // 路由反向编译
    ruleUrl(url, query = {}) {
        if(!routes[url]) return url;
        url = routes[url].url;

        if(~url.indexOf(':')) {
            url = url.replace(/\:([^/]+\(.*\)|[^/]+)/g, (match, key) => {
                key = key.split(/[(.]/);
                const k = key[0];
                if(k in query) {
                    const value = query[k];
                    delete query[k];
                    return querystring.escape(value) 
                           + (!key[1] || ~key[1].indexOf(')') ? '' : '.' + key[1]);
                } else if(this.ctx.params && k in this.ctx.params) {
                    return querystring.escape(this.ctx.params[k]);
                }
                return match;
            });
        }

        return url;
    }
}
```

**使用示例：**

```javascript
// 智能解析
this.$url.build('user/list');
// → /user/list

this.$url.build('detail', {id: 123});
// → /index/detail?id=123

// 路由命名
this.$url.build(':article', {id: 123});
// → /article/123.html

// 带域名
this.$url.build('index', {}, '.html', true);
// → http://127.0.0.1:3000/index/index.html
```

## 工作流程

### 完整请求流程

```
1. 用户发起 HTTP 请求
   ↓
2. Koa 接收请求，进入中间件链
   ↓
3. Storage 中间件
   - 记录 APP_TIME、APP_VERSION
   - 创建 AsyncLocalStorage 上下文
   - 初始化 $ 和 _ 加载器
   ↓
4. 用户自定义中间件（如果有）
   ↓
5. 异常处理中间件
   - try { await next() }
   - catch 错误并记录日志
   - 调试模式输出详细错误
   - 记录请求日志（方法、状态、URL、耗时）
   ↓
6. 静态资源中间件（如果配置）
   - 匹配 public 目录文件
   - 匹配则直接返回，不继续
   ↓
7. koa-body 中间件（如果配置）
   - 解析 POST 数据
   - 处理文件上传
   ↓
8. 路由中间件
   - 匹配自定义路由（routes.js）
   - 匹配默认路由（/:APP/:CONTROLLER/:ACTION）
   - 设置 ctx.APP、ctx.CONTROLLER、ctx.ACTION
   ↓
9. run() 执行
   - 设置 storage 上下文（APP、CONTROLLER、ACTION）
   - 加载配置到 storage
   - 验证应用/控制器/方法合法性
   ↓
10. 控制器处理
    - 实例化控制器类
    - 加载控制器中间件
    - 执行 _init()
    - 执行 action()
    - 执行 _end()
    ↓
11. 响应输出
    - $show() 直接输出
    - $fetch() 渲染模板
    - $redirect() 重定向
    - $success()/$error() 提示跳转
    ↓
12. 中间件链返回（洋葱模型）
    ↓
13. 请求结束，AsyncLocalStorage 自动清理
```

### 类实例化流程

```javascript
// 示例：在控制器中使用 this.$model.user

this.$model.user
    ↓
1. Ctx Proxy 拦截 $model 访问
    ↓
2. 查找顺序：
   a. app/model/ 目录？✓ 找到
   b. 创建 Proxy 代理该目录
    ↓
3. 访问 .user 属性
    ↓
4. Loader Proxy 拦截
    ↓
5. 查找 app/model/user.js
    ↓
6. 检测是否为 Class？✓ 是
    ↓
7. 自动 new User(ctx) 创建实例
    ↓
8. 缓存实例到 infoStore.instance
    ↓
9. 返回实例，后续访问直接返回缓存
    ↓
10. 调用方法：this.$model.user.getUserInfo({id: 1})
```

## 特性总结

### 核心优势

1. **创新的自动加载机制**
   - Proxy + AsyncLocalStorage 组合
   - 无需手动 import/require
   - 懒加载 + 单例化
   - 智能路径查找（应用 → 公共 → 框架）
   - 路径判断结果缓存，性能优秀

2. **完整的 MVC 架构**
   - 经典三层架构（Model-View-Controller）
   - 模仿 ThinkPHP5，PHP 开发者友好
   - 三级中间件系统

3. **强大的数据库支持**
   - 多数据库驱动（MySQL/SQLite/MongoDB）
   - 完整的查询构造器
   - 链式调用，代码优雅
   - 内置查询缓存
   - 事务支持
   - 字段过滤防注入

4. **灵活的路由系统**
   - 自定义路由配置
   - 默认路由（智能解析）
   - 路由命名和反向编译
   - 支持动态参数
   - 支持三种路由类型（controller/middleware/view）

5. **开发体验优秀**
   - 完整的 JSDoc 类型提示
   - 自动生成 types 文件
   - 调试模式友好
   - 热更新支持（types 文件）

6. **轻量级设计**
   - 核心代码精简
   - 按需加载模块
   - 静态资源可选

### 技术亮点

| 特性 | 实现方式 | 优势 |
|------|---------|------|
| 自动加载 | Proxy 拦截 | 无需手动导入，代码简洁 |
| 生命周期管理 | AsyncLocalStorage | 请求隔离，自动清理 |
| 类单例化 | Loader 缓存实例 | 节省内存，避免重复创建 |
| 路径缓存 | typeStore Map | 避免重复文件检查 |
| 查询构造器 | 链式调用 + 参数绑定 | 防止 SQL 注入，代码优雅 |
| 模板引擎 | art-template | 性能好，语法简单 |
| 路由系统 | @koa/router | 成熟稳定，功能丰富 |

## 编码规范

### 命名规范

- **类名**：大驼峰（PascalCase）
  ```javascript
  class UserController { }
  class UserModel { }
  ```

- **方法名**：小驼峰（camelCase）
  ```javascript
  async getUserInfo() { }
  async userList() { }
  ```

- **私有方法**：下划线前缀
  ```javascript
  _validateData() { }
  _formatResult() { }
  ```

- **控制器文件名**：小写下划线
  ```
  user_controller.js
  article_list.js
  ```

### 目录命名

- 应用目录可自定义（app、admin、api 等）
- controller、middleware 目录名固定
- model、view 等目录名可配置
